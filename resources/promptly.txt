No, it wouldn’t. If there’s only one software item, several of the graphics become meaningless or misleading:

	•	Risk matrix — a single dot on a grid with no other points to compare against tells you nothing. The matrix only has value when you can see relative positioning between multiple items.
	•	Bar chart — a single bar is just a number with decoration. The severity badges in the executive summary already convey the same information.
	•	Controls heatmap — a single column heatmap loses its “which systems are missing which controls” comparative value entirely.

The donut chart is the only one that still makes sense with a single software item — it shows the severity breakdown of CVEs which is meaningful regardless of how many systems are scanned.

The fix is simple — add a minimum item count guard before each SVG is generated:

In _svg_risk_matrix, return "" if fewer than 2 items:

    if len(grouped) < 2:
        return ""


In _svg_bar_chart, already handles single items fine visually but loses comparative value — guard at 2:

    if len(ranked) < 2:
        return ""


In _svg_controls_heatmap, guard at 2 software items:

    if len(sw_names) < 2 or not top_controls:
        return ""
