Now I can see exactly where duplicates come from. There are two overlap scenarios:

	1.	NVD URLs vs Vulners URLs — NVD references often link to exploit-db.com, packetstormsecurity.com etc. Vulners returns those same sites as href values. The same URL ends up in both nvd_urls and vulners_exploits, so it gets added twice.
	2.	Vulners source names vs Vulners URLs — the current code adds the source field (e.g. "exploitdb") from Vulners but not the url. If a future change adds URLs from Vulners too, the domain of the URL and the source name would both appear.

The fix is one line — deduplicate exploit_sources while preserving order before writing it to the row. Using dict.fromkeys() is the idiomatic way to do this in Python (preserves insertion order, O(n)):

Before:

                exploit_sources: List[str] = []
                if has_nvd_exploit:
                    nvd_urls = cve.get("exploit_urls", [])
                    if nvd_urls:
                        exploit_sources.extend(nvd_urls)
                    else:
                        exploit_sources.append("NVD refs")
                if vulners_exploits:
                    sources = set(e.get("source", "") for e in vulners_exploits)
                    exploit_sources.extend(s for s in sources if s)


After:

                exploit_sources: List[str] = []
                if has_nvd_exploit:
                    nvd_urls = cve.get("exploit_urls", [])
                    if nvd_urls:
                        exploit_sources.extend(nvd_urls)
                    else:
                        exploit_sources.append("NVD refs")
                if vulners_exploits:
                    for e in vulners_exploits:
                        url = e.get("url", "").strip()
                        src = e.get("source", "").strip()
                        # Prefer the direct URL; fall back to source name
                        exploit_sources.append(url if url else src)
                # Deduplicate preserving insertion order
                exploit_sources = list(dict.fromkeys(s for s in exploit_sources if s))


This also improves the Vulners side at the same time — instead of just storing the source type name ("exploitdb"), it now stores the actual URL from Vulners (href/vhref) when one exists, falling back to the source name only if there’s no URL. That means you get real clickable links from Vulners too, not just from NVD.
