The problem

In _markdown_body_to_html, the table cell handler at line ~359 does:

row_html = "".join(f"<td>{inline(escape(c))}</td>" for c in cells)


escape() runs html.escape() on the raw cell content, turning <br> into &lt;br&gt; before it ever reaches the browser.

The fix

There are two options. The cleaner one is to skip escaping on cell content that already contains HTML tags, by checking before escaping:

            close_list()
            cells = [c.strip() for c in line.strip("|").split("|")]
            # Separator row (e.g. |---|---|)
            if all(re.match(r"^:?-+:?$", c) for c in cells if c):
                if not table_header_done and in_table:
                    # Replace last <tr> as a header row
                    last = out.pop()
                    header_tr = last.replace("<tr>", "<thead><tr>").replace("</tr>", "</tr></thead><tbody>")
                    out.append(header_tr)
                    table_header_done = True
                continue
            if not in_table:
                out.append('<table>')
                in_table = True
            # If a cell already contains HTML tags (e.g. <br>, <a>), pass it
            # through inline() without escaping — it was pre-built as safe HTML.
            def _cell(c: str) -> str:
                if re.search(r"<[a-zA-Z/]", c):
                    return f"<td>{inline(c)}</td>"
                return f"<td>{inline(escape(c))}</td>"
            row_html = "".join(_cell(c) for c in cells)
            out.append(f"<tr>{row_html}</tr>")

The re.search(r"<[a-zA-Z/]", c) check detects whether the cell content contains an HTML tag. If it does, it skips escape() and passes the content directly to inline() so the <br> tags render as line breaks. All other cells continue to be escaped normally.
